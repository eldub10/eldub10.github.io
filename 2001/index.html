 PORTFOLIO

BridgePort


BridgePort™ is Supertracks' last-element streaming technology that delivers locally repeatable music in a single stream from cached, secure music files. This delivery method creates a cost-effective way to deliver mainstream music over the Internet, while providing higher, CD-quality music playback. It also enables new advertising and promotion models incorporating demographic targeting and detailed tracking.

The BridgePort platform is a modular design utilizing a framework architecture based on the Component Object Model (COM). Component DLLs were created in C++ using ATL. User interface components were created in Visual Basic and DHTML.

The platform was designed to accommodate prospective features and security requirements; support for new content usage requirements can be added dynamically. BridgePort's asynchronous data collection method provides all of the advantages of a rich central database without creating the latency problems typically associated with back-end data processing. The BridgePort architecture contains multiple levels of security and encryption technologies designed to protect copyright holders and to comply with the legal requirements of the Digital Millennium Copyright Act (DMCA). The BridgePort platform extends the content distribution network to its final element — the home PC. Utilizing the processing power and local drive capacity in today's PCs, BridgePort is well suited to address the capacity, administration, and bandwidth issues facing conventional streaming systems.

This product was distributed in a public beta called Jam Player in conjunction with Portland radio station Jammin 95.5, and also in another program with HipHopSite called XTuner.

RTrack


RTrack™ is Supertracks' Business-to-Business digital distribution of music (DDM) system that allows online retailers the ability to sell music from Supertracks' digital warehouse. This system can be described as a multi-step process that begins with the automated delivery of the product catalog. The data is delivered in XML format, using HTTP, to retailers for incorporation into their databases. Retailers make these products available for purchase using their existing e-commerce systems. When a retailer's customer purchases an item, a purchase transaction is sent to Supertracks (in XML format) using HTTPS with mutual authentication. Supertracks handles these requests in real-time using ISAPI extensions and COM components that record the transaction in a SQL Server database using OLEDB. A license is generated for the purchased music and a link is sent back to the retailer, who displays it to the end user. When the end user clicks on the link in their browser, the music is downloaded from Supertracks' digital warehouse to the end user's computer and unlocked using the associated license.

Strategic partners in this project were Microsoft, Preview Systems, eMusic, HMV, Virgin Megastores, EMI, and TVT.

SIMNetwork


This prototype for a tile-based real-time educational strategy game was to be used in information technology training. The game simulates computer networking environments and allows users to experiment with installation, configuration, management, troubleshooting, and other networking processes. Additional components of the game also simulate the network operating system's desktop environment. The game architecture consists of a graphics engine programmed in C++ using DirectX, a logic engine that draws scenarios from a database of operating system rules, software properties, and hardware configurations, and a desktop simulator consisting of components built from resource templates that mimic a graphical user interface. This provides an extremely realistic simulation of the operating system environment.
